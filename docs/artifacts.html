Software Design and Engineering

Event Tracking Android Mobile Application

Originally created in CS 360 Mobile Architecture and Programming. This artifact was enhanced by improving login logic, input validation, and error handling.

Skills Demonstrated: mobile application design, validation, maintainability, user workflow improvement

Original Code Enhanced Code

The selected artifact is an Event Tracking Android mobile application originally created in CS 360: Mobile Architecture and Programming. The application allows users to log in, create events, and manage event details using a local SQLite database.

This artifact was selected because it demonstrates core software engineering principles, including application structure, user interaction, and data persistence. Specific components such as the login workflow, event creation screens, and database integration showcase the ability to build a functional, user-focused application. Enhancements improved the artifact by redesigning the login workflow, adding persistent login functionality with logout capability, implementing input validation, and refining navigation between screens. Updates to files such as LoginActivity.java, EventsActivity.java, EventDetailsActivity.java, and UserDatabase.java strengthened both usability and code structure.

The enhancement process demonstrated skills in debugging, improving user experience, and applying best practices in application design. Challenges included managing activity navigation, ensuring data consistency between UI and database layers, and resolving issues such as incorrect field mappings and non-functional buttons. Feedback was incorporated to improve validation logic and user flow, resulting in a more stable and intuitive application.

This artifact supports multiple course outcomes, including designing and evaluating computing solutions, implementing software engineering practices, and communicating technical decisions effectively. While collaboration was limited in this project, the design decisions reflect consideration for maintainability and real-world development environments. The final result is a more polished, functional, and user-centered application.

Algorithms and Data Structures

Maze Reinforcement Learning Agent

Originally developed in CS 370 Current and Emerging Technologies. This artifact was enhanced by refining reward shaping, tuning epsilon decay, and adding performance evaluation to better demonstrate learning behavior.

Skills Demonstrated: reinforcement learning, algorithm analysis, performance evaluation, parameter tuning

Original Code Enhanced Code

The artifact is a reinforcement learning-based maze-solving agent developed in CS 370: Current and Emerging Technologies. The agent uses a Q-learning approach with a neural network to navigate a maze and reach a goal state.

This artifact was selected because it highlights algorithmic thinking, problem-solving, and the application of machine learning concepts. Key components include the Q-training function, model architecture, and reward system, which collectively demonstrate an understanding of decision-making algorithms and optimization strategies. Enhancements focused on improving training efficiency and model performance by adjusting parameters such as epsilon decay, reward shaping, training duration, and batch size.

The enhancement process involved significant experimentation and tuning. Challenges included slow training times, unstable loss behavior, and inconsistent learning results. Improvements such as reducing reward magnitude, adjusting exploration rates, and limiting training steps resulted in better early performance, including increased win rates and reduced average steps per episode. These results demonstrate measurable progress compared to the original implementation, even though the model did not reach full convergence within the available time.

This artifact demonstrates achievement of course outcomes related to designing algorithmic solutions and evaluating trade-offs in implementation. It also highlights the importance of iterative testing and performance analysis in machine learning workflows. While the model did not achieve a 100% success rate, the enhancements show clear improvement and provide a strong foundation for continued optimization.

Databases

Event Tracking Android Mobile Application

This artifact was also used for the databases category. Enhancements focused on secure password reset design, improved schema support, and stronger data handling.

Skills Demonstrated: SQLite database design, secure data handling, schema enhancement, authentication workflow

Original Code Enhanced Code

The artifact is the database component of the Event Tracking application, also originally developed in CS 360. It uses SQLite to store user and event data, supporting core application functionality.

This artifact was selected because it demonstrates essential database skills, including schema design, data validation, and secure data handling. Key components such as user authentication, event storage, and query execution highlight the ability to design and interact with a relational database system. Enhancements focused on improving data integrity, refining schema usage, and strengthening validation and security practices.

Enhancements included improving query parameterization to prevent SQL injection, refining database access methods in UserDatabase.java, and ensuring proper handling of user credentials and event data. Additional improvements addressed issues with retrieving and displaying correct data, ensuring that event details and categories were stored and retrieved accurately.

The enhancement process reinforced the importance of clean database design and secure coding practices. Challenges included debugging incorrect query results, managing database connections, and ensuring consistency between stored data and UI display. Feedback was used to improve validation logic and ensure that the database layer supported the application reliably.

This artifact demonstrates achievement of course outcomes related to implementing computing solutions using appropriate tools and techniques, as well as developing a security mindset. While more advanced database features such as normalization and scalability could be further explored, the enhancements significantly improved the reliability and security of the application’s data layer.
